  <action>
Create src/water.js from scratch. The file has no imports or exports — all symbols are global, matching the established pattern in game-state.js / player.js / platforms.js.

Write these sections in order:

SECTION 1 — Constants (tune-friendly, one declaration per line):
```javascript
const FLOOD_BASE_SPEED  = 30;    // px/s at level 1 — tune during playtesting
const FLOOD_ACCEL       = 1.5;   // px/s per second — linear speed increase within a level
const FLOOD_LEVEL_SCALE = 0.4;   // each level multiplies base by (1 + (level-1) * scale)
const WAVE_AMPLITUDE    = 10;    // px — sine crest height
const WAVE_FREQUENCY    = 0.04;  // radians per pixel — controls wave width
const WAVE_SPEED        = 2.5;   // radians per second — controls animation pace
const IFRAME_DURATION   = 1.8;   // seconds of invincibility after damage hit
const FLASH_DURATION    = 0.4;   // seconds the red overlay is visible
```

SECTION 2 — water object:
```javascript
const water = {
  waterY:      700,  // world Y of wave surface — starts 60px below canvas bottom (cameraY=0)
  floodSpeed:  FLOOD_BASE_SPEED,
  waveTime:    0,
  iframeTimer: 0,
  flashTimer:  0,
};
```

SECTION 3 — resetWater(level):
Resets waterY to 700, floodSpeed to FLOOD_BASE_SPEED * (1 + (level - 1) * FLOOD_LEVEL_SCALE),
waveTime / iframeTimer / flashTimer all to 0.

SECTION 4 — takeDamage():
Decrements GameState.lives by 1. Sets water.iframeTimer = IFRAME_DURATION and
water.flashTimer = FLASH_DURATION. If GameState.lives <= 0, calls
saveHighScore(GameState.score) then sets GameState.phase = GamePhase.GAMEOVER.
The iframeTimer guard is NOT checked here — it is checked by the caller (updateWater).

SECTION 5 — updateWater(dt):
- Tick timers: if > 0, subtract dt and clamp to 0 via Math.max(0, ...) for both iframeTimer and flashTimer.
- Rise: water.waterY -= water.floodSpeed * dt
- Accelerate: water.floodSpeed += FLOOD_ACCEL * dt
- Animate: water.waveTime += WAVE_SPEED * dt
- Collision: const playerBottom = player.y + player.h; const collisionY = water.waterY - WAVE_AMPLITUDE;
  if (playerBottom >= collisionY && water.iframeTimer <= 0) { takeDamage(); }

SECTION 6 — renderWater(ctx):
Must be called inside the world-space ctx.save/translate block — ctx is already translated by cameraY.
Draw a filled polygon:
- fillStyle: 'rgba(30, 144, 255, 0.75)'
- beginPath
- moveTo(0, water.waterY + Math.sin(water.waveTime) * WAVE_AMPLITUDE)
- Loop x from 1 to 480 inclusive: lineTo(x, water.waterY + Math.sin(x * WAVE_FREQUENCY + water.waveTime) * WAVE_AMPLITUDE)
- lineTo(480, water.waterY + 2000)  — far sentinel, fills body
- lineTo(0, water.waterY + 2000)
- closePath then fill

Add a short comment above renderWater explaining that it must be called before ctx.restore() in the
world-space block.

Add a comment at the top of the file: "// src/water.js — rising flood water: object, physics, collision, rendering"
and note dependencies: "// Depends on: GameState, GamePhase, saveHighScore (game-state.js), player (player.js)"
