# Plan 03-02: Procedural Platforms — Generation, Crumbling State Machine, Rendering

## Objective

Replace the single hardcoded starter platform in `platforms.js` with a full procedural generation system. The level's platforms are all generated upfront when `resetPlatforms()` is called. Crumbling platforms have a three-state machine (`intact` → `cracked` → `gone`) driven by the `dt` accumulator, not `setTimeout`. Normal and crumbling platforms are visually distinct.

## Context

@src/platforms.js — current file to replace (read before editing)
@src/player.js — physics constants (JUMP_VELOCITY = -700, GRAVITY = 980, player start y = 528)
@src/game-state.js — GameState.levelGoalY written here; GameState.level read here

Physics constraint verified from research:
- Max jump height = 700² / (2 × 980) ≈ 250 px theoretical
- Safe max gap = 250 × 0.80 = 200 px hard limit
- Recommended gap = 120 px (48% of theoretical max, comfortable play)

Current `platforms.js` state:
- Single object array with one hardcoded platform `{ x: 190, y: 560, w: 100, h: PLATFORM_H }`
- `resetPlatforms()` rebuilds that single platform
- `updatePlatforms(dt)` is empty
- `checkPlatformCollisions()` works correctly (do NOT modify the 4-condition one-way check)
- `renderPlatforms(ctx)` renders everything with one `fillStyle`

3. Replace `resetPlatforms()` entirely with `generateLevelPlatforms()` called from inside `resetPlatforms()`. The public function signature stays `resetPlatforms()` so existing calls in `game-state.js` (resetGame, startNextLevel) do not need changing:

```javascript
function resetPlatforms() {
  generateLevelPlatforms(GameState.level);
}

function generateLevelPlatforms(level) {
  platforms.length = 0;

  const levelHeight = LEVEL_BASE_HEIGHT + (level - 1) * 500;

  // Always start with a fixed starter platform directly under spawn point
  platforms.push({
    x: 190, y: 560, w: 100, h: PLATFORM_H,
    type: 'normal', state: 'intact', crumbleTimer: 0,
  });

  // Store the level goal world Y in GameState so main.js can check and draw it
  GameState.levelGoalY = PLAYER_START_Y - levelHeight;

  // Generate platforms in upward slots from the player start position
  const slotCount = Math.floor(levelHeight / GAP_PX);
  for (let i = 1; i <= slotCount; i++) {
    const worldY = PLAYER_START_Y - i * GAP_PX;
    const w      = PLATFORM_MIN_W + Math.random() * (PLATFORM_MAX_W - PLATFORM_MIN_W);
    const x      = 20 + Math.random() * (480 - w - 40); // 20px margin each side
    const type   = Math.random() < CRUMBLE_CHANCE ? 'crumble' : 'normal';

    platforms.push({
      x:            Math.floor(x),
      y:            Math.floor(worldY),
      w:            Math.floor(w),
      h:            PLATFORM_H,
      type,
      state:        'intact',
      crumbleTimer: 0,
    });
  }
}
```

The level height formula: Level 1 = 2000 px (~16 platforms at 120px gap). Level 2 = 2500 px (~20 platforms). This gives a natural difficulty ramp without touching flood speed (Phase 4 handles that).

### Task 2: Crumbling State Machine — updatePlatforms and renderPlatforms

<action>
1. Replace `updatePlatforms(dt)` with the crumble timer accumulator:
```javascript
function updatePlatforms(dt) {
  // Advance crumble timers for all cracked platforms (backward loop for safe splice)
  for (let i = platforms.length - 1; i >= 0; i--) {
    const p = platforms[i];
    if (p.type === 'crumble' && p.state === 'cracked') {
      p.crumbleTimer += dt * 1000; // dt is seconds; timer is in ms
      if (p.crumbleTimer >= CRUMBLE_DELAY_MS) {
        platforms.splice(i, 1); // remove gone platform
      }
    }
  }
}
```

2. In `checkPlatformCollisions()`, add crumble state transition inside the collision block, directly after the bounce snap. The 4-condition one-way check itself is unchanged:
```javascript
if (overlapX && wasAbove && nowBelow && movingDown) {
  player.y  = p.y - player.h;
  player.vy = JUMP_VELOCITY;

  // Crumble: intact → cracked on first landing. Removal is deferred to updatePlatforms().
  if (p.type === 'crumble' && p.state === 'intact') {
    p.state        = 'cracked';
    p.crumbleTimer = 0;
  }
}
```

Note: The loop in `checkPlatformCollisions` currently uses `for (const p of platforms)`. This must be changed to a regular index loop when using splice, BUT the splice happens in `updatePlatforms` (not here), so `for...of` is still safe in `checkPlatformCollisions`. Only `updatePlatforms` needs the backward index loop.

3. Replace `renderPlatforms(ctx)` with type- and state-aware rendering:
```javascript
function renderPlatforms(ctx) {
  for (const p of platforms) {
    if (p.type === 'crumble') {
      // Visual distinction: red intact crumble, orange cracked crumble
      ctx.fillStyle = p.state === 'cracked' ? '#e67e22' : '#e74c3c';
    } else {
      ctx.fillStyle = '#27ae60'; // green — normal platform
    }
    ctx.fillRect(Math.floor(p.x), Math.floor(p.y), p.w, p.h);
  }
}
```

Color summary:
- Normal platform: green `#27ae60`
- Crumbling platform (intact): red `#e74c3c` — visually distinct from normal
- Crumbling platform (cracked): orange `#e67e22` — shows damage before disappearing
