# Plan 03-01: GameState Foundation — Level Fields, Reset Functions, LocalStorage

## Objective

Extend `game-state.js` with the complete data layer for Phase 3: a fourth GamePhase constant (`LEVEL_COMPLETE`), level-specific fields on GameState, two distinct reset functions (`resetGame` vs `startNextLevel`), and a LocalStorage high score system. Everything in this plan is pure data and functions — no rendering, no input, no physics.

This plan creates the contracts that Plans 03-02 and 03-03 build against.

### Task 2: Add startNextLevel and LocalStorage High Score Functions

<files>
  src/game-state.js
</files>

<action>
Add the following four things to the bottom of `game-state.js`, after the existing `resetGame()` function:

1. **`startNextLevel()` function** — increments level, resets score-related state, does NOT touch lives:
```javascript
function startNextLevel() {
  GameState.level           += 1;
  GameState.score            = 0;
  GameState.cameraY          = 0;
  GameState.maxHeightReached = 9999;
  GameState.phase            = GamePhase.PLAYING;
  // GameState.lives is intentionally NOT reset — lives persist across levels
  resetPlayer();
  resetPlatforms(); // also sets GameState.levelGoalY for the new level
}
```

2. **LocalStorage key constant**:
```javascript
const HS_KEY = 'soggymoggy_highscore';
```

3. **`loadHighScore()` function** — reads from localStorage with parseInt and try/catch:
```javascript
function loadHighScore() {
  try {
    const raw = localStorage.getItem(HS_KEY);
    GameState.highScore = raw !== null ? parseInt(raw, 10) : 0;
    if (isNaN(GameState.highScore)) GameState.highScore = 0;
  } catch (e) {
    GameState.highScore = 0; // graceful fallback for Firefox file:// and private mode
  }
}
```

4. **`saveHighScore(score)` function** — only writes if new score beats the current best:
```javascript
function saveHighScore(score) {
  if (score > GameState.highScore) {
    GameState.highScore = score;
    try {
      localStorage.setItem(HS_KEY, String(score));
    } catch (e) {
      // Storage unavailable — silent fallback
    }
  }
}
```

5. **Call `loadHighScore()` once** at the bottom of game-state.js (after the function definition), so it runs when the script loads and before any other script reads GameState.highScore.

Important: `startNextLevel()` calls `resetPlatforms()`, which is defined in `platforms.js`. This is safe at runtime because classic script tags load synchronously and platforms.js loads after game-state.js. The comment in the existing `resetGame()` already documents this pattern.
</action>
