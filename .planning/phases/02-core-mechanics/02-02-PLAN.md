## Camera Math (Reference)

| Symbol | Meaning |
|--------|---------|
| `cameraY` | World Y at the top of the visible screen area |
| `ctx.translate(0, -cameraY)` | Shifts world objects UP on screen by cameraY pixels |
| negative `cameraY` | Camera shows world area above Y=0 — correct for tall vertical worlds |
| `SCROLL_THRESHOLD` | `canvas.height * 0.4 = 256px` — player kept at this screen Y position |

**Scroll rule:** When `player.y - cameraY < 256` (player is above threshold), set `cameraY = player.y - 256`. Only apply if this DECREASES cameraY (prevents scroll-back when player falls).

**Fall rule:** Player world Y > `cameraY + canvas.height` → off the bottom of the visible screen → GAMEOVER (Phase 4: subtract life instead).

## Tasks

### Task 1 — Add `updateCamera()` to `src/main.js`

**Goal:** Camera follows player's peak height and never scrolls back.

**Add the `updateCamera()` function** to `src/main.js` (after the `renderHUD` function):

```javascript
// ── Camera ───────────────────────────────────────────────────────────────────
function updateCamera() {
  const SCROLL_THRESHOLD = canvas.height * 0.4; // 256px — player held at 40% from top

  // Scroll up: only update if player has climbed above the threshold (cameraY can only decrease)
  const newCameraY = player.y - SCROLL_THRESHOLD;
  if (newCameraY < GameState.cameraY) {
    GameState.cameraY = newCameraY;
  }

  // Track maximum height reached (lower Y = higher in world — stores minimum Y value)
  if (player.y < GameState.maxHeightReached) {
    GameState.maxHeightReached = player.y;
  }
}
```

**Why `newCameraY < GameState.cameraY`:** This is the "never scroll back" gate. As the player falls (y increases), `newCameraY` increases too — which is always ≥ the already-set `cameraY`, so the condition fails and camera stays fixed. Camera only moves when the player reaches a new high point.

**Commit:** `feat(02-02): add updateCamera() — follows player peak, never scrolls back`

### Task 2 — Wire camera and fall detection into `update()` in `src/main.js`

**Goal:** Call `updateCamera()` each PLAYING frame; detect fall-off-bottom → GAMEOVER.

**Change the `case GamePhase.PLAYING:` block** in `update(dt)`:

Replace:

```javascript
case GamePhase.PLAYING:

  updatePlatforms(dt);

  // Phase 4 will add: updateWater(dt)

```

With:
```javascript
case GamePhase.PLAYING:
  updatePlayer(dt);
  updatePlatforms(dt);
  checkPlatformCollisions();
  updateCamera();

  // Fall-off-bottom detection: player below visible area → game over
  // Phase 4 replaces this with: lives-- + respawn mechanic
  if (player.y > GameState.cameraY + canvas.height) {
    GameState.phase = GamePhase.GAMEOVER;
  }

  // Phase 4 will add: updateWater(dt)
  break;
```

**Order matters:** `updateCamera()` is called AFTER `checkPlatformCollisions()`. This ensures the camera position is computed from the player's final (post-collision) Y position for the frame — not a mid-flight intermediate value.

**Fall detection note:** `GameState.cameraY + canvas.height` is the world Y of the bottom of the visible screen. With `cameraY=-56`, visible bottom = `584`. A player at `y=585` is off screen.

**Verification for Task 2:**

### Task 3 — Update `renderHUD()` in `src/main.js` to show height debug info

**Goal:** During PLAYING phase, show current height (for debugging Phase 2 physics) alongside the existing phase indicator.

**Change `renderHUD()`** — add height display to the PLAYING state:

```javascript
function renderHUD() {
  ctx.fillStyle = '#000000';
  ctx.font      = '16px monospace';

  // Phase indicator (debug — always visible)
  ctx.fillText('Phase: ' + GameState.phase, 8, 20);

  if (GameState.phase === GamePhase.PLAYING) {
    // Debug: height display — shows how high the player has climbed
    // Phase 3 will replace with proper score formatting
    const heightPx = Math.max(0, 528 - GameState.maxHeightReached); // pixels above start
    ctx.fillStyle = '#ffffff';
    ctx.fillText('Height: ' + heightPx + ' px', 8, 42);
  }

  if (GameState.phase === GamePhase.START) {
    ctx.fillStyle = '#ffffff';
    ctx.font      = '20px monospace';
    ctx.fillText('SOGGY MOGGY', 160, 300);
    ctx.font = '14px monospace';
    ctx.fillText('Press Arrow / A-D to start', 110, 340);
  }

  if (GameState.phase === GamePhase.GAMEOVER) {
    ctx.fillStyle = '#ffffff';
    ctx.font      = '20px monospace';
    ctx.fillText('GAME OVER', 170, 300);
    ctx.font = '14px monospace';
    ctx.fillText('Press Arrow / A-D to restart', 105, 340);
  }
}
```

**Note on height formula:** `528 - GameState.maxHeightReached`
- Player starts at world y=528. Lower world Y = higher in the game.
- `maxHeightReached` stores the minimum world Y the player has reached.
- When player is at world y=300 (climbed 228px): height = 528 - 300 = 228px ✓
- `Math.max(0, ...)` clamps to 0 so it never shows negative on game start.

**Verification for Task 3:**
- `Height: X px` appears on screen during PLAYING phase
- Height value increases as the player bounces higher
- Height value does NOT decrease when the player falls back down

**Commit:** `feat(02-02): add height debug display to HUD`

## Human Verification Steps

These require a running browser:

**Test A — Camera scroll visible:**
Start the game. After a few bounces the player should rise above the starting platform view. The platform should appear to sink toward the bottom of the canvas and eventually scroll off-screen below.

**Test B — Camera never scrolls back:**
Let the camera scroll up (player climbs several bounces high). Then let the player fall. The canvas background should NOT scroll back down — the camera holds at its highest position.

**Test C — Height counter:**
Watch the `Height: X px` HUD text. It should increase steadily as the player bounces higher. When the player falls back toward the platform level, the counter should NOT decrease.

**Test D — GAMEOVER from fall:**
This is hard to trigger naturally in Phase 2 (player always bounces from the starter platform). To test: open DevTools console and type `GameState.cameraY = -500` then watch — the starting platform is now off-screen above the view, and the player (at world y≈528) is below the visible area (visible bottom = -500+640 = 140), so GAMEOVER fires immediately.
